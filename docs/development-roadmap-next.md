# 泉峰AI数据中心运维平台后续开发计划

更新时间：2026-06-18

## 当前判断

当前平台已经具备 v0.1 原型基础：机柜总览、U 位大图、资产台账、虚拟服务器、进出管理、变更管理、连线展示、告警中心、Webhook 接入、AI 助手、Agent API、统一持久化基座、SQLite/Tauri 表结构基座和测试体系。

下一步不再继续堆新页面，优先把“真实数据进来、页面看得到、AI 查得到、操作有审计、可以备份恢复”做成闭环。

## v0.1.0 收口目标：真实运维可演示

1. **Webhook 告警真实接入**
   - 任意 Webhook 内容都要进入告警中心。
   - 标准字段自动匹配设备、机柜和位置。
   - 非标准字段保留原始内容，方便确认卓豪、Prometheus 或自定义系统到底发了什么。
   - 告警列表默认显示描述列，未匹配设备明确显示“未匹配设备”。

2. **机柜总览与定位体验**
   - AI 查询、告警定位、资产定位后，让设备尽量移动到屏幕中心。
   - U 位大图保持 42U/正背面切换/多排多柜浏览稳定。
   - 529 数据中心的柜型颜色、3D 视图颜色和 2D 总览保持一致。

3. **资产台账稳定化**
   - Excel 导入模板固定化。
   - 物理服务器、存储、网络、安全、精密空调、列头柜、配线柜字段分层。
   - 资产详情窗口统一查看、编辑、实例关系、变更记录、二维码。

4. **AI 查询闭环**
   - AI 能查询资产、虚拟机、进出记录、变更记录、告警、机柜位置。
   - 平台内部问题优先走工具/API 查询真实数据。
   - 通用运维问题走当前启用模型回答。
   - 每次回答记录工具调用、模型、耗时、证据和定位目标。

5. **基础验收**
   - `pnpm test` 全量通过。
   - 关键页面冒烟：机柜总览、资产管理、告警中心、进出管理、变更管理、系统设置。
   - 局域网访问和 Webhook 地址生成可用。

## v0.2.0 数据底座：从浏览器本地存储走向统一后端

1. **SQLite 统一数据源**
   - 逐步把资产、机房、机柜、设备、告警、进出、变更、连线、AI 轨迹迁移到 SQLite。
   - 浏览器 localStorage 只保留 UI 偏好和临时状态。
   - 提供首次启动迁移脚本，把旧数据安全导入 SQLite。

2. **Tauri/Rust 后端命令**
   - 完成设备、机柜、告警、进出、变更、连线的 CRUD 命令。
   - 统一返回错误结构，前端显示友好提示。
   - 敏感字段走密钥链或加密存储，不进入普通备份。

3. **本地 HTTP 服务**
   - 将 `vite.config.ts` 中的 API、中间件、Agent API、Webhook 逻辑迁到 `local-service/server`。
   - 开发环境和生产环境使用同一套接口合同。
   - 预留后续多人部署模式。

4. **备份恢复**
   - 项目 JSON 导入导出保留非敏感业务数据。
   - SQLite 快照备份、自动备份、恢复前安全检查。
   - 导出时扫描 token、password、secret、api key 等敏感字段。

## v0.3.0 运维闭环：让平台真正服务日常工作

1. **告警处理闭环**
   - 告警状态：未读、处理中、已解决、已恢复。
   - 支持处理说明、附件、照片、维修结论。
   - 告警可关联资产、机柜、进出记录、变更记录。

2. **变更管理深化**
   - 记录上架、下架、接线、硬件扩容、系统安装、故障维修。
   - 每条变更可关联设备、机柜、责任人、时间、附件。
   - AI 可查询和录入变更，但写入前必须确认。

3. **进出管理深化**
   - 支持 Excel 导入、手动录入、AI 录入。
   - 支持按机房、单位、人员、设备、故障原因、日期检索。
   - 与资产维修历史联动。

4. **连线管理深化**
   - 资产详情中维护端口。
   - 连线管理展示拓扑视图、表格视图、端口视图。
   - 支持交换机端口占用、对端设备、线缆编号、导出接线表。

5. **报表中心**
   - 设备清单、机柜容量、告警趋势、变更统计、进出统计。
   - 月度运维报告和 AI 摘要。

## v0.4.0 AI Agent：从查询助手到受控运维智能体

1. **Agent 角色和能力配置**
   - 可编辑角色定义、长期记忆、Skill、知识库。
   - 区分只读账号和管理员账号能力边界。

2. **工具调用体系**
   - 资产查询、机柜定位、告警查询、进出查询、变更查询、连线查询。
   - 写入工具包括资产新增/修改、进出录入、变更录入、告警处理。
   - 所有写入工具需要确认、审批和审计。

3. **澄清与交互式录入**
   - 当用户意图不明确时，AI 主动给出选项。
   - 需要补字段时，AI 弹出结构化表单。
   - 用户确认后再执行写入。

4. **知识库与附件**
   - 支持日志、文档、图片附件上传。
   - 大文件分块、摘要、索引、检索。
   - AI 回答引用知识库和平台真实数据。

5. **执行轨迹**
   - 记录模型、工具、参数、结果、耗时、错误、证据链。
   - 系统设置中统一查看 AI 审计和非 AI 操作审计。

## v0.5.0 企业化部署与集成

1. **账号权限**
   - 管理员、普通账号、只读账号。
   - 模块级只读/修改权限。
   - API Key 区分只读和写入 scope。

2. **监控系统适配**
   - 卓豪监控 API。
   - Prometheus API。
   - 告警去重、恢复、关联资产和机柜。

3. **CMDB/MCP 同步**
   - CMDB 同步资产。
   - ZStack MCP 同步虚拟服务器。
   - 同步前预览差异，同步后记录审计。

4. **多人部署**
   - 后端服务化。
   - SQLite 可切换 PostgreSQL。
   - 角色权限、审计日志、备份策略、部署文档。

5. **开源发布**
   - 完善 README、安装文档、贡献规范、示例数据、许可证说明。
   - 保证不包含公司真实密钥、账号、敏感资产。

## 当前下一步优先级

1. Webhook 告警真实接入增强。
2. AI 查询进出/变更/告警结果稳定化。
3. 机柜定位居中和告警定位体验优化。
4. 把本地 API 从 `vite.config.ts` 继续拆到 `local-service/server`。
5. SQLite 读写命令逐表接入。

## 每轮开发验收规则

1. 先写失败测试，再写实现。
2. 至少运行相关单测。
3. 影响页面时做浏览器冒烟。
4. 不能上传 `.env.local`、日志、数据库、token、password、secret、api key。
5. 功能完成后提交 Git，并按需要推送 GitHub。
