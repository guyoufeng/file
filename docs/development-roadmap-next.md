# 泉峰AI数据中心运维平台后续开发计划

更新时间：2026-06-04

## 当前阶段目标

v0.1.0 继续以“可用、专业、可演示”为核心，优先把机柜总览、资产、告警、进出、变更、AI 查询和基础写入闭环做稳。后续再逐步切换到 Tauri + SQLite、本地服务端和多人部署。

## 下一阶段建议

1. 数据持久化统一层  
   状态：已完成 v0.1 基座。进出、变更、虚拟服务器、连线、连线视图、知识库、Skill、Agent 执行记录、消息网关会话已进入 `qf-ai-dcim.data.*`。API Key、账号凭据、Webhook Token、网关 Token 已纳入统一清单但默认不导出。

2. SQLite/Tauri 数据层  
   状态：已完成表结构基座。新增 `0002_operational_agent_data.sql`，覆盖进出、变更、虚拟机、连线、Agent 轨迹、知识库、Skill、网关、API Key、凭据、Webhook 和表格列配置。Tauri 项目导出已追加非敏感运行数据；逐表写入命令下一阶段继续补。

3. 本地服务模式  
   状态：部分完成。前端已有只读/写入 Agent API 清单、Webhook URL 生成和外部演示模式；真正统一数据源的本地 HTTP 服务仍需后续接入 SQLite。

4. 权限模型深化  
   状态：部分完成。账号权限、AI 写入权限、Agent API Key read/write scope 已具备；后端强制校验和审计中间件下一阶段完成。

5. 表格个性化  
   状态：已完成 v0.1 表格基座。资产、虚拟服务器、进出、变更、告警已支持列显示和排序配置；表格配置已预留 SQLite 表。

6. AI 澄清与确认  
   状态：已完成第一版。AI 遇到“录入/新增/修改”但目标模块不明确时，会提供资产、进出、变更选项。高风险动作的执行前后差异确认仍需下一阶段深化。

7. AI 工具化写入  
   状态：部分完成。资产新增/更新、进出记录、变更记录已工具化，并修复“变更误写成资产”的路由问题。告警、连线写入工具下一阶段补齐。

8. Agent 执行记录  
   状态：已完成 v0.1 基座。Agent 执行轨迹、用时、事件、模型、附件元数据和定位目标已记录，并纳入项目备份。

9. 知识库与附件  
   状态：部分完成。知识库条目已纳入统一数据和备份；附件解析已有基础能力。100MB 级日志/图片的分块索引、缓存和检索增强仍需下一阶段继续做。

10. 消息网关  
    状态：平台侧已完成基座。支持微信、企业微信、钉钉配置、配对二维码、独立会话和共享 Skill。真实个人微信扫码登录必须由 Hermes/iLink 或官方企业微信/钉钉适配器完成。

11. Webhook 告警接入  
    状态：部分完成。Webhook 配置、URL、Token、告警入库和设备匹配已具备；卓豪/Prometheus 字段映射模板、去重和AI处理建议下一阶段完善。

12. 连线管理增强  
    状态：已完成 v0.1 展示基座。连线记录、交换机端口摘要、拓扑保存视图已进入统一数据和备份；真实端口模板和资产详情端口维护下一阶段深化。

13. 变更闭环  
    状态：部分完成。变更可关联机房、机柜、设备、业务IP，并支持AI自然语言录入。与进出、告警、连线的自动联动下一阶段继续做深。

14. 备份恢复  
    状态：已完成 v0.1 JSON 闭环。项目JSON导入导出已包含非敏感运行数据，并通过安全扫描排除密钥、Token、密码字段；SQLite快照和自动备份下一阶段做。

15. 测试体系  
    状态：持续完成。本轮新增统一持久化、项目导出、SQLite schema、Agent API 权限清单测试，并继续保留全量测试、构建和冒烟验证流程。

## 优先开发顺序

1. 完成本地统一数据层到 SQLite 的替换。
2. 把只读/写入 API 权限校验移到统一服务层。
3. 完成 AI 工具调用、澄清、确认和审计闭环。
4. 完成 Webhook 告警真实接入。
5. 完成连线管理和资产详情接口拓扑增强。
6. 再做真实消息网关适配器和多人部署。
