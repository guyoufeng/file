# 泉峰AI数据中心运维平台后续开发计划

更新时间：2026-06-04

## 当前阶段目标

v0.1.0 继续以“可用、专业、可演示”为核心，优先把机柜总览、资产、告警、进出、变更、AI 查询和基础写入闭环做稳。后续再逐步切换到 Tauri + SQLite、本地服务端和多人部署。

## 下一阶段建议

1. 数据持久化统一层  
   把仍直接访问浏览器 localStorage 的模块逐步迁到 `createPersistentCollection`，所有集合统一进入 `qf-ai-dcim.data.*` 命名空间。

2. SQLite/Tauri 数据层  
   在 `src-tauri` 建立 SQLite schema、迁移脚本和 Rust command，把前端持久化接口替换为 Tauri 调用。

3. 本地服务模式  
   增加可选本地 HTTP 服务，让浏览器演示、外部 Agent API、Webhook 接入和未来多人访问共用同一数据源。

4. 权限模型深化  
   把账号权限从前端配置迁到后端校验，AI 写入、API 写入、Webhook 写入都必须经过统一权限和审计。

5. 表格个性化  
   所有业务表格支持列显示、列顺序、分页、筛选保存和导出字段选择，配置按用户保存。

6. AI 澄清与确认  
   当 AI 写入目标不明确时先给选项；高风险动作必须二次确认，并记录用户选择、执行前数据和执行后数据。

7. AI 工具化写入  
   资产、进出、变更、告警、连线统一封装成工具，AI 只通过工具执行，不直接改数据对象。

8. Agent 执行记录  
   每轮问答保存问题、模型、工具、用时、证据、轨迹、结果和用户确认，支持按设备、IP、模块、时间搜索。

9. 知识库与附件  
   文本、日志、Excel、图片附件进入知识库索引；大文件先分块摘要，再让模型按证据回答。

10. 消息网关  
    平台侧先完成回调、配对、会话隔离和审计；真实微信、企业微信、钉钉登录需要独立 Gateway Adapter。

11. Webhook 告警接入  
    完成卓豪/Prometheus webhook 解析、告警去重、状态更新、定位设备和 AI 处理建议。

12. 连线管理增强  
    资产详情中维护物理接口，连线管理用于展示拓扑、端口表和保存视图，后续支持交换机端口模板。

13. 变更闭环  
    变更记录关联资产、机柜、进出记录、告警和连线，设备详情中能查看完整变更历史。

14. 备份恢复  
    JSON 导入导出保留，SQLite 后增加数据库快照、自动备份、恢复预览和敏感字段排除。

15. 测试体系  
    每个核心模块保留单元测试；每轮功能完成后跑 `pnpm test`、`pnpm build`、`pnpm test:smoke`。

## 优先开发顺序

1. 完成本地统一数据层到 SQLite 的替换。
2. 把只读/写入 API 权限校验移到统一服务层。
3. 完成 AI 工具调用、澄清、确认和审计闭环。
4. 完成 Webhook 告警真实接入。
5. 完成连线管理和资产详情接口拓扑增强。
6. 再做真实消息网关适配器和多人部署。
