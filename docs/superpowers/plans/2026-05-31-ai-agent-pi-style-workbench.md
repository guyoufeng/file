# 泉峰 AI Agent Pi-style 工作台实施计划

## 目标

把平台内嵌 AI 助手从“查询入口”升级为可审计、可扩展、可配置的小型智能体：通用问题走当前大模型，平台事实问题走只读 API/数据库工具，角色、记忆、Skill、知识库、凭据和外部工具都能在平台里维护。所有执行步骤都要留下记录，便于未来外部 Agent、CMDB、MCP、监控工具接入。

## Pi Agent 对照原则

1. Agent 生命周期可观察：启动、接收问题、规划、工具选择、工具执行、模型总结、审计写入、结束。
2. 每一轮会话有运行记录：问题、回答、模型、工具、耗时、事件、附件元数据和定位对象。
3. 工具是受控资源：平台事实只走只读工具；未来写操作必须单独授权。
4. Skill、记忆、知识库是 Agent 资源：可在设置中维护，也可通过自然语言更新。
5. 凭据不进入模型提示词：模型只看到脱敏目录，真正密钥只给工具执行器使用。
6. 对用户展示的是“可审计执行过程”，不是不可验证的模型私有推理文本。

## 具体任务清单

1. 阅读 Pi Agent 的 Harness、事件流、Session、Skill 和 Observability 设计。
2. 固化本平台 Agent v0.1 能力边界：只读平台工具 + 通用模型问答 + 可配置资源。
3. 新增 Agent 运行记录数据结构，保存 runId、sessionId、问题、答案、工具、模型、状态、耗时和事件。
4. 新增事件计时工具，把每个 Agent 事件转换成带 elapsedMs / durationLabel 的可展示轨迹。
5. AI 助手发送消息后立即显示“执行过程”，每秒刷新耗时。
6. AI 回复完成后，把完整轨迹写入当前会话，重新打开会话仍能看到。
7. Agent 轨迹中显示工具名、数据源、阶段耗时和失败原因。
8. 新增 Agent 执行记录面板，可搜索历史问题、工具、模型、失败原因。
9. Agent 执行记录详情展示问题、回答摘要、模型、工具、起止时间、总耗时和完整事件。
10. 写入 Agent 运行记录时同步关联定位对象，便于未来跳转机柜/设备。
11. 新增 CMDB/MCP 工具接入配置服务，保存类型、名称、Endpoint、Token、启用状态和备注。
12. CMDB/MCP Token 在提示词中脱敏，只显示“已保存/未保存”。
13. 设置页新增“CMDB / MCP 工具接入”配置面板，支持保存、测试、启用/停用、删除。
14. Agent Prompt 注入已启用的 CMDB/MCP 工具目录，让模型知道可用工具边界。
15. 清理 Agent 配置页暂不可用的天气查询和网页搜索灰色开关。
16. 保留通用问答和监控 API Skill 记忆开关，避免用户被无效能力干扰。
17. 强化通用问题路由：机房维护建议、硬件维修、阈值建议等直接走大模型。
18. 强化平台问题路由：IP、计算机名、SN、责任人、机柜、机房、告警、审计、虚拟机都走平台工具。
19. 编写 20 个平台问题矩阵测试，覆盖物理机、机柜、IP、责任人、资产、告警、审计、虚拟机。
20. 编写 20 个通用问题矩阵测试，覆盖温湿度、ECC、CPU/内存阈值、巡检、维护流程等。
21. 编写 Agent 运行记录单元测试，验证保存、搜索、清理和耗时格式。
22. 编写 CMDB/MCP 工具配置单元测试，验证脱敏和测试结果。
23. 更新 AI 助手冒烟测试，验证执行过程、秒级耗时、轨迹和通用问答。
24. 更新设置页冒烟测试，验证 Agent 执行记录、CMDB/MCP 工具接入、无天气/网页搜索灰色项。
25. 运行单元测试，修复所有失败。
26. 运行类型检查和构建，修复类型/UI 编译问题。
27. 运行浏览器冒烟测试，重点验证 AI 助手、设置页、机柜定位不回退。
28. 浏览器手工验证 3-5 个真实交互问题，确认 UI 行为符合预期。
29. 整理当前完成情况、测试证据和下一阶段建议。
30. 后续规划：流式输出、真正工具调用循环、写操作权限、CMDB/MCP 实际同步、监控告警 AI 分析。

## 验证目标

- 单元测试覆盖 Agent 运行记录、工具接入、问题路由矩阵。
- 构建通过 `pnpm build`。
- 冒烟测试覆盖 AI 助手、设置页、只读 API、机柜定位。
- 页面上看不到无效天气/网页搜索灰色 Skill。
- AI 助手每轮回答都能展开执行过程，并显示秒级耗时。
